  • Static sites

    These simple sites use the bare minimum of code while attaining to the modern standards of design to reduce the loading time for the user and the development time for the programmer. Their content is always the same, so they're exempt of the complexity of servers and databases, lowering their cost of operation. On the other hand, these sites are the least flexible as they are unable to provide the user any form of content creation or account system. These sites are ideal for informing customers about your company.

    Technologies: HTML, CSS, Bootstrap, MaterializeCSS, Netlify

  • Javascript powered sites

    In the cases where the application doesn't need a server but has a complex user interface, we can recur to Javascript to make the product more interactive. Whether your website needs to do calculations, fetch information from an external source or handle highly complex animations, this type of site will suit your needs. And with the help from Firebase, it is possible for your users to create accounts and content.

    Technologies: HTML, CSS, Javascript, Svelte, Firebase

  • Server rendered apps

    This classical website architecture involves a web server that interacts with a database to provide the customers with the ability to save different types of data and to authenticate for additional benefits. This type of sites also open the door to services like newsletters, scheduled tasks and content management systems.

    Technologies: Django, PostgreSQL, HTML, CSS

  • Single page applications

    When the interface grows too complex but the application still needs the benefits of a server, the solution is usually to have a Javascript powered site that communicates with the server via an API, which makes the database elements available over the internet. They are called single page applications because they don't need to refresh when navigating inside them. This sites have the shortcoming of scoring poorly in the search engine rankings because the algorithms that evaluate them can't access their content, thus they are not as likely to appear as the first result when the users search for it.

    Technologies: Django, PostgreSQL, HTML, CSS, Javascript, Svelte

  • Server rendered Javascript apps

    The solution to the above problem is to create another server in charge of rendering the interface for the indexing algorithms. This way, the site is more favored by the browsers but it also reduces the loading time for the users

    Technologies: Django, PostgreSQL, HTML, CSS, Javascript, Svelte, Sapper
